# Part 7: Future Possibilities (Roadmap V2+)

> **Status:** Brainstorm Result
> **Date:** November 18, 2025
> **Topic:** Ideas beyond MVP

Этот раздел содержит идеи, которые не входят в MVP, но закладывают вектор развития проекта.

## 7.1. Privacy & Security 2.0

- **AES Encryption:** Возможность защитить инвойс паролем. Данные в URL шифруются симметричным ключом. Плательщик должен ввести пароль (полученный в секретном чате), чтобы расшифровать детали платежа.
- **ZK Proofs:** (Research) Генерация доказательства оплаты (Receipt) с использованием Zero-Knowledge, чтобы подтвердить факт оплаты третьей стороне без раскрытия суммы.

## 7.2. Расширенные платежи

- **Cross-Chain Swaps:** Интеграция виджета (Li.Fi / Jumper / Squid Router).
  - _Сценарий:_ Инвойс в USDC на Arbitrum. У плательщика только ETH на Optimism. Виджет позволяет оплатить в один клик с автоматическим бриджем и свапом.
- **Escrow Contracts:** Опциональный режим "Secure Deal". Средства лочатся в смарт-контракте и высвобождаются только после подтверждения выполнения работы (или по истечению времени).

## 7.3. Масштабирование данных

- **IPFS Offloading:** Для длинных ТЗ или прикрепленных файлов. Данные загружаются в IPFS, в URL передается только CID.
- **Decentralized Sync:** Синхронизация истории между устройствами пользователя с использованием Ceramic Network или GunDB (по подписи кошелька).

## 7.4. Экосистемные интеграции

- **Telegram Mini App:** Нативная версия приложения внутри Телеграма для выставления счетов прямо в чате.
- **Gnosis Safe App:** Интеграция в интерфейс мультисиг-кошельков для удобной оплаты счетов командами DAO.
- **Accounting Export:** Экспорт восстановленной истории транзакций в форматы CSV/PDF, совместимые с крипто-бухгалтериями (CoinTracking, Koinly).

# Part 3: User Experience & Design

> **Status:** Brainstorm Result
> **Date:** November 18, 2025
> **Topic:** Visual Identity, User Flows & Interface

## 3.1. Визуальный стиль (Visual Identity)

Проект должен транслировать ощущения: **Надежность, Технологичность, Чистота**. Мы избегаем "игривого" крипто-стиля (memes, pixel art) в пользу "Fintech Corporate".

### 3.1.1. Палитра и Тема

- **Default Mode:** Dark Mode (Глубокий серый/угольный, например `Zinc-950`). Это стандарт де-факто для Web3 инструментов.
- **Accent Color:** **Electric Violet (`#7C3AED`)** — Tailwind Violet-600.
  - Используется только для первичных действий (кнопка "Create", "Pay").
  - Яркий, современный, выделяется на dark mode.
  - Ассоциируется с технологичностью и футуризмом.
- **Semantic Colors:**
  - Success: Emerald (`#10B981`) — для статуса "Paid".
  - Warning: Amber (`#F59E0B`) — для статуса "Processing" или "Wrong Network".
  - Error: Rose (`#F43F5E`) — для ошибок валидации.

### 3.1.2. Типографика

- **UI / Headings:** **`Geist Sans`**
  - Современный гротеск от Vercel.
  - Оптимизирован для Next.js (автоматический font optimization).
  - Отличная читабельность на экранах любого размера.
- **Data / Numbers:** **`Geist Mono`**
  - Моноширинный шрифт из той же семьи Geist.
  - Используется для адресов кошельков, сумм, токенов, Chain ID.
  - _Важно:_ Включена поддержка `tabular-nums` (моноширинные цифры) — разряды выравниваются вертикально в таблицах и списках.

**Преимущества единой семьи Geist:**
- Визуальная когерентность (Sans и Mono из одной семьи).
- Меньше font loading overhead.
- Лучшая совместимость с Next.js font optimization.

### 3.1.3. Визуализация сетей (Ambient Context)

Интерфейс адаптируется под выбранную сеть, чтобы дать подсознательный контекст пользователю.

- **Механика:** На странице `/pay` и в превью `/create` меняется глобальный фон и акцентные цвета.
- **Темы:**
  - _Arbitrum:_ Глубокий синий градиент (`bg-slate-950` + blue glow).
  - _Optimism:_ Красноватое свечение (`bg-slate-950` + red glow).
  - _Polygon:_ Фиолетовая дымка.
  - _Ethereum:_ Строгий серый/монохром.

## 3.2. Сценарий Создателя (Route: `/create`)

**Цель:** Создать инвойс максимально быстро, с возможностью предпросмотра.

### 3.2.1. Интерфейс (Split Screen)

- **Desktop:** Экран разделен на две части.
  - _Left (Editor):_ Скроллящаяся форма ввода данных.
  - _Right (Preview):_ Закрепленная ("Sticky") область, отображающая инвойс так, как его увидит клиент (HTML-версия будущего PDF). Обновляется в реальном времени (Reactive).
- **Mobile:** Использование табов `[Edit]` и `[Preview]`.

### 3.2.2. Ключевые взаимодействия

1.  **Smart Address Input:** При вводе адреса кошелька или ENS, справа появляется сгенерированный аватар (Blockies/Jazzicon). Это визуальный "Checksum" — если аватарка знакомая, значит адрес верный.
2.  **Token Selector:** Выпадающий список с поиском. Поддержка "Paste Address" для кастомных токенов с автоматической подтяжкой символа и decimals через RPC.
3.  **Developer Tip Checkbox:** Чекбокс "Support Devs (Add 1% Tip)" позволяет добавить 1% к сумме платежа в пользу разработчиков. Опция доступна при создании инвойса.
4.  **Branding Field:** Только текстовое название компании (text-only input). Нет возможности загрузки логотипа, чтобы избежать битых картинок и проблем с хостингом изображений.
5.  **Generation:** Кнопка "Create Link" копирует URL в буфер и показывает модальное окно с QR-кодом и кнопками шаринга (Telegram, Twitter, Copy).

## 3.3. Сценарий Плательщика (Route: `/pay`)

**Цель:** Оплатить без лишних кликов и сомнений.

### 3.3.1. Интерфейс (The Receipt Card)

Страница выглядит как цифровой чек. Никаких лишних меню.

- **Header:** Название компании отправителя (text-only, без логотипа).
- **Hero Section:** Огромная сумма к оплате и статус.
  - **Pretty Print Display:** Основная сумма отображается красиво округленной: `1,000.00 USDC` (крупным шрифтом).
  - **Exact Amount:** Мелким шрифтом ниже подписано: _"Payment amount: 1,000.000042"_ — точная сумма с Magic Dust для честности и прозрачности.
- **Details:** Таблица услуг (свернутая по умолчанию на мобильных).
- **Footer:** "Powered by VoidPay" (обязательный watermark).

### 3.3.2. Web3 Взаимодействие (The "Pay" Button)

Кнопка оплаты — это "умный" компонент с несколькими состояниями:

1.  **State: Disconnected** -> Кнопка "Connect Wallet".
2.  **State: Wrong Network** -> Кнопка "Switch to Arbitrum" (Polite Switch - пользователь нажимает кнопку для переключения сети, не агрессивное автопереключение).
3.  **State: Insufficient Funds** -> Кнопка заблокирована, подпись "Not enough USDC".
4.  **State: Ready** -> Кнопка "Pay 1,000.00 USDC" (отображается Pretty Print сумма, точная сумма с Magic Dust видна в мелком тексте выше).
    - _Логика:_ Если это ETH — вызывается `sendTransaction`. Если токен — `writeContract (transfer)`. Appove не нужен (так как P2P).
5.  **State: Processing** -> Спиннер, ссылка на Explorer.
6.  **State: Success** -> Конфетти, кнопка меняется на "Download Receipt".

### 3.3.3. Элементы доверия и безопасности

- **Report Abuse:** В самом низу страницы (в футере, серым цветом) должна быть кнопка "Report Abuse". Это стандарт индустрии для UGC (User Generated Content) платформ.
- **Disclaimer:** Под кнопкой оплаты мелкий текст: _"You are sending funds directly to [Address]. VoidPay is not an intermediary."_

## 3.4. Мобильный опыт (Mobile UX)

- **Deep Linking:** При нажатии "Connect Wallet" на мобильном устройстве должен открываться нативный кошелек (Metamask/Rainbow) через протокол WalletConnect (используем Web3Modal для обработки этого).
- **In-App Browsers:** Если сайт открыт внутри Telegram/Instagram, показываем предупреждение: _"For best experience, open in Safari/Chrome"_ (так как инжекшн кошелька в вебвью часто работает некорректно).
